# ブラウザだけで本格開発——「Claude Code on the web（ClaudeCowork）」の活用方法

**ローカル環境を一切用意せずに、AIがコードを書き、テストを走らせ、プルリクエストまで作成する。そんな未来の開発スタイルが「Claude Code on the web（通称：ClaudeCowork）」によって現実になっている。本記事では、このサービスの概要から具体的な使い方まで、わかりやすく解説する。**

調査日：2026-03-17

---

## ClaudeCoworkとは何か

「Claude Code on the web」（以下、ClaudeCowork）は、Anthropicが提供するAI搭載のコーディングアシスタント「Claude Code」のブラウザ版だ。

通常のClaude Codeはターミナル（コマンドを入力する画面）にインストールして使うが、ClaudeCoworkはブラウザで[claude.ai/code](https://claude.ai/code)にアクセスするだけで利用できる。

仕組みとしては、Anthropicが管理するクラウド上の仮想マシン（インターネット上の専用コンピュータ）でClaude Codeが動く。ユーザーは自分のパソコンに何もインストールする必要がなく、GitHubのリポジトリ（ソースコードの保存場所）を指定するだけでタスクを実行できる。

2026年3月時点では「リサーチプレビュー」（正式リリース前の試験提供）段階にあり、以下のプランのユーザーが利用できる。

- **Proユーザー**
- **Maxユーザー**
- **Teamユーザー**
- **Enterpriseユーザー**（プレミアムシートまたはChat + Claude Codeシートを持つ場合）

## ClaudeCoworkが特に役立つ場面

公式ドキュメントによると、ClaudeCoworkは以下のシーンで威力を発揮する。

### 1. 時間がかかる定型作業を任せる

バグ修正、テストの作成、コードの整形など、「やらなければいけないが後回しにしがちな作業」を任せることができる。Claudeが自律的に作業を進めるため、その間に別の仕事ができる。

例えば「認証モジュールのテストを書いて、実行して、失敗したら修正して」と指示するだけで、一連の作業を自動で行ってくれる。

### 2. 並行して複数の作業を進める

ClaudeCoworkは複数のセッション（作業の一連のやり取り）を同時に動かせる。「バグAの修正」「ドキュメントの更新」「リファクタリング（コードの整理）」を別々のセッションで同時進行させることが可能だ。

ターミナルから以下のように一気に複数タスクを起動することもできる。

```bash
claude --remote "認証のバグを修正して"
claude --remote "APIドキュメントを更新して"
claude --remote "ロガーを構造化出力に変更して"
```

### 3. 手元にないリポジトリでの作業

自分のパソコンにダウンロードしていないコードでも作業できる。GitHubのリポジトリを指定するだけで、Claudeがクラウド上でコードを取得して作業を行う。

### 4. バックエンドのコード変更

テスト駆動開発（テストを先に書いてからコードを実装する手法）にも対応しており、バックエンドの変更を安全に行いやすい。

## 実際の使い方：ステップバイステップ

### 基本的な始め方

1. [claude.ai/code](https://claude.ai/code) にアクセスする
2. GitHubアカウントと連携する
3. Claude GitHub Appをリポジトリにインストールする
4. 使用する環境（クラウド環境）を選ぶ
5. 作業内容を自然な言葉で入力して送信する
6. 変更内容を確認して、プルリクエストを作成する

### ターミナルとの連携

すでにClaude Codeのターミナル版を使っている場合、`--remote` フラグを使うとターミナルからWebセッションを起動できる。

```bash
claude --remote "src/auth/login.tsの認証バグを修正して"
```

このコマンドを実行すると、クラウド上でタスクが実行され始める。ローカルでの作業を続けながら、Claudeがクラウドで処理を進める形だ。

### 変更内容の確認（差分ビュー）

ClaudeCoworkには、Claudeが変更したファイルを視覚的に確認できる「差分ビュー」機能がある。変更前後のコードが並んで表示されるため、プルリクエストを作成する前に内容を確認して、必要に応じてコメントで修正を依頼できる。

### セッションの移行（テレポート機能）

ブラウザで作業を始めて、途中からターミナルで続けたい場合は `/teleport`（または `/tp`）コマンドを使う。逆に、ターミナルで進行中の作業をスマートフォンやブラウザから確認・継続することもできる。

セッションの移行には以下の条件が必要だ。

- コミットされていない変更がないこと
- 同じリポジトリのチェックアウトで実行すること
- 同じClaude.aiアカウントで認証されていること

## セキュリティと安全性

クラウド上で自分のコードが実行されることへの心配もあるだろう。ClaudeCoworkのセキュリティ対策を確認しておこう。

### 分離された仮想マシン

各セッションは、Anthropicが管理する独立した仮想マシン上で動く。他のユーザーの作業との混同は起きない。

### ネットワークアクセスの制限

デフォルトでは、アクセスできるインターネット上のサービスが制限されている。npmやPyPIなど主要なパッケージレジストリ（ライブラリの配布サービス）には接続できるが、それ以外のサイトへのアクセスは制限される。設定によって「制限なし」や「インターネット遮断」も選べる。

### 認証情報の保護

GitHubの認証トークン（ログイン用の鍵）などの重要な認証情報は、仮想マシン内には置かれない。専用のセキュリティプロキシ（中継サーバー）を通じて安全に処理される。

---

## 現時点での制限事項

便利なClaudeCoworkだが、現時点ではいくつかの制限がある。

- **GitHubのみ対応**：GitLabや他のサービスにホストされたコードには使えない
- **同一アカウントが必要**：WebセッションをローカルPCに移行する際は、同じアカウントで認証されている必要がある
- **リサーチプレビュー段階**：正式リリース前のため、機能や仕様が変わる可能性がある

## より効果的に使うためのポイント

### CLAUDE.mdを活用する

リポジトリのルート（一番上のフォルダ）に `CLAUDE.md` というファイルを置くと、Claudeはセッション開始時にそれを読み込む。コーディングスタイルや注意事項、プロジェクト特有のルールを書いておくと、毎回説明する手間が省ける。

### セットアップスクリプトを設定する

クラウド環境でClaude Codeが起動する前に実行されるスクリプト（プログラム）を設定できる。プロジェクト固有のツールや依存関係（必要なライブラリ）を自動でインストールするよう設定しておくと、毎回手動で準備する必要がなくなる。

### 計画してから実行する

複雑なタスクでは、まずローカルで「計画モード（Plan Mode）」を使ってアプローチを練り、その後Webセッションで実行するという流れが効果的だ。

```bash
# まずローカルで計画を立てる
claude --permission-mode plan

# 計画ができたらWebで実行
claude --remote "docs/migration-plan.mdの移行計画を実行して"
```

### セッションを共有する

TeamやEnterpriseプランでは、作業中のセッションをチームメンバーと共有できる。コードレビューや引き継ぎに活用できる機能だ。

## まとめ

Claude Code on the web（ClaudeCowork）は、ブラウザだけで本格的なAI支援開発ができるサービスだ。ローカル環境の構築が不要で、複数タスクの並行処理や、手元にないリポジトリへの対応など、従来の開発ワークフローを大きく変える可能性を持っている。

現在はリサーチプレビュー段階だが、セキュリティ面での配慮もしっかりなされており、本格的な業務利用への準備が整いつつある。GitHubのみの対応など制限はあるものの、開発者の生産性を大幅に高めるツールとして今後の発展が期待される。

「AIに作業を任せている間に、人間はより創造的な仕事に集中する」——そんな開発スタイルへの転換を、ClaudeCoworkは後押ししてくれるだろう。
