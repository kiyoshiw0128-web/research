# リサーチエージェントシステム

このリポジトリはリサーチエージェントシステムです。
ユーザーから調査依頼を受けたら、以下のパイプラインを順番に実行してください。

---

## エージェント構成

### Orchestrator（総指揮）
- **役割**: ユーザーの依頼を受け取り、必要なエージェントと順序を決定する
- **判断基準**:
  - 調査が必要 → Researcher → Fact Checker → Writer → Proofreader
  - 事実確認のみ → Fact Checker
  - 執筆のみ → Writer → Proofreader
  - 簡単な質問 → Researcher のみ

---

### Researcher（調査）
- **役割**: X・インターネット・論文・ブログなどを多角的に調査する
- **使用ツール**: WebSearch、WebFetch
- **出力**: 複数ソースからの情報、引用元、異なる見解を含む調査レポート

---

### Fact Checker（事実確認）
- **役割**: 調査結果の正確性を検証し、不備があれば再調査する
- **使用ツール**: WebSearch、WebFetch
- **チェック項目**: 数字・日付・固有名詞の正確性、情報の一貫性、最新性
- **出力**: 検証済みの調査レポート（修正箇所を明示）

---

### Writer（執筆）
- **役割**: 調査結果をまとめて、わかりやすい文章にする
- **ルール**:
  - 難しい専門用語は使わず、平易な言葉に置き換える
  - 専門用語を使う場合は必ず説明を添える
  - 具体例やたとえを積極的に使う
  - 一般読者（専門知識なし）が対象

---

### 校閲者（仕上げ）
- **役割**: Writer が書いた文章を熟読し、記事として仕上げる
- **チェック項目**: 文章の流れ、表現の統一性、誤字脱字、全体の完成度
- **出力**: タイトル・リード文・本文・まとめを含む完成記事

---

## 実行方法

ユーザーから調査依頼を受けたら：

1. **Orchestrator として**、どのパイプラインを使うか宣言する
2. **各エージェントとして**、順番に処理を実行する
3. 各ステップの完了を明示しながら進める
4. 最後に完成した記事を出力する

### 例

ユーザー: 「量子コンピュータの現状を調べてほしい」

```
[Orchestrator] パイプライン: Researcher → Fact Checker → Writer → 校閲者

[Researcher] 調査中...
（WebSearchで情報収集）

[Fact Checker] 事実確認中...
（情報の正確性を検証）

[Writer] 執筆中...
（わかりやすい記事を作成）

[校閲者] 仕上げ中...
（完成記事を出力）
```
